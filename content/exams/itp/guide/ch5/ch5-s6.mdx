# システム企画

システム企画では、システム化の目的、範囲、費用対効果、調達方法を明確にします。RFI、RFP、提案評価、要件定義へつながる上流工程です。

## 1. 企画の流れ

| 手順 | 内容 |
|---|---|
| 現状分析 | 業務上の問題や要求を把握する |
| 課題設定 | 解決すべき課題を明確にする |
| システム化構想 | 目的、対象範囲、期待効果を整理する |
| 費用対効果分析 | 投資に見合う効果があるか確認する |
| 調達計画 | 開発方法や委託先選定の方針を決める |

## 2. 調達に関する文書

| 文書 | 内容 |
|---|---|
| RFI | 情報提供依頼。技術動向や事例を集める |
| RFP | 提案依頼。要件を示して提案を求める |
| 提案書 | ベンダが提示する実現方法、費用、体制 |
| 契約書 | 範囲、責任、納期、費用、権利などを定める |

## 3. 要件の種類

| 種類 | 内容 |
|---|---|
| 機能要件 | システムが実行すべき機能 |
| 非機能要件 | 性能、可用性、セキュリティ、運用性など |
| 業務要件 | 業務上実現したいこと |
| 制約条件 | 予算、納期、既存システム、法令など |

## 4. 提案評価

- 要件を満たしているか。
- 費用が妥当か。
- 実現可能なスケジュールか。
- セキュリティや運用への配慮があるか。
- ベンダの実績や体制が十分か。

## 試験での注意点

RFIは情報収集、RFPは具体的な提案依頼です。費用の安さだけで選ぶのではなく、要件適合、品質、運用、リスクも評価します。
