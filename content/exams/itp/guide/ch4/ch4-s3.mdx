# プロジェクトマネジメント

プロジェクトマネジメントは、期限と予算がある一時的な活動を成功させるために、範囲、品質、費用、納期、リスク、関係者を管理する考え方です。

## 1. QCDとスコープ

| 観点 | 内容 |
|---|---|
| Quality | 品質 |
| Cost | 費用 |
| Delivery | 納期 |
| Scope | 作業範囲 |

QCDは互いに関係します。短納期で範囲を広げれば、費用や品質に影響が出ます。

## 2. 計画と進捗管理

| 手法 | 内容 |
|---|---|
| WBS | 作業を細かく分解する |
| ガントチャート | 作業期間や進捗を帯で表す |
| アローダイアグラム | 作業の順序関係を矢印で表す |
| クリティカルパス | 遅れると全体納期に影響する最長経路 |

## 3. リスク管理

| 手順 | 内容 |
|---|---|
| 識別 | 起こり得るリスクを洗い出す |
| 分析 | 発生確率と影響度を評価する |
| 対応 | 回避、低減、移転、受容を決める |
| 監視 | 状況の変化を確認する |

## 4. ステークホルダ管理

ステークホルダは、プロジェクトに影響を与える、または影響を受ける関係者です。利用者、経営者、開発者、運用担当、外部委託先などが含まれます。

## 試験での注意点

WBSは作業を分解する手法、ガントチャートは期間を見える化する手法、クリティカルパスは納期に直結する経路です。似た図の役割を区別しましょう。
