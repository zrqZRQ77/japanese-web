# ソフトウェア開発管理技術

ソフトウェア開発管理技術では、開発をどのような進め方で管理するかを学びます。開発モデル、アジャイル、DevOps、構成管理、変更管理が重要です。

## 1. 開発モデル

| モデル | 内容 | 向いている場面 |
|---|---|---|
| ウォーターフォール | 工程を順番に進める | 要件が明確で変更が少ない |
| プロトタイピング | 試作品を作り確認しながら進める | 画面や操作性を確認したい |
| スパイラル | 開発と評価を繰り返す | リスクを段階的に減らしたい |
| アジャイル | 短い期間で開発と改善を繰り返す | 変更に柔軟に対応したい |

## 2. アジャイルとスクラム

| 用語 | 内容 |
|---|---|
| イテレーション / スプリント | 短期間の開発サイクル |
| プロダクトバックログ | 実現したい機能や改善項目の一覧 |
| デイリースクラム | 日々の短い進捗確認 |
| レトロスペクティブ | 振り返りによる改善 |

## 3. DevOpsと継続的改善

DevOpsは、開発担当と運用担当が連携し、システムを継続的に改善する考え方です。自動テスト、自動デプロイ、監視を組み合わせて、変更を安全かつ迅速に反映します。

## 4. 構成管理と変更管理

| 管理 | 内容 |
|---|---|
| 構成管理 | ソースコード、設定、文書などの版を管理する |
| 変更管理 | 変更の影響を確認し、承認して実施する |
| バージョン管理 | 変更履歴を記録し、必要に応じて戻せるようにする |

## 試験での注意点

ウォーターフォールは計画管理しやすい一方で変更に弱く、アジャイルは変更に強い一方で継続的な利用者との確認が重要です。どちらが常に優れているという問題ではありません。
